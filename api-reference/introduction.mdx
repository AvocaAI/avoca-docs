## Overview

The Avoca AI API provides programmatic access to our AI-powered communication and scheduling platform for home service businesses. This REST API enables developers to integrate Avoca's voice AI, scheduling, messaging, and analytics capabilities into their applications.

## Base URL

All API requests should be made to:

```
https://api.avoca.ai/v1
```

## Authentication

All API requests require authentication using an API key. Include your API key in the request headers:

```bash
Authorization: Bearer YOUR_API_KEY
X-Workspace-ID: YOUR_WORKSPACE_ID
```

<Note>
To obtain your API credentials, log in to your [Avoca Dashboard](https://app.avoca.ai) and navigate to Settings â†’ API Keys.
</Note>

## Rate Limiting

To ensure fair usage and system stability, the following rate limits apply:

- **1,000** requests per minute per API key
- **10,000** requests per hour per workspace

Rate limit information is included in all API responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 998
X-RateLimit-Reset: 1642521600
```

## Request Format

All requests must include the following headers:

```bash
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY
X-Workspace-ID: YOUR_WORKSPACE_ID
```

## Response Format

All API responses return JSON data with consistent formatting:

### Success Response

```json
{
  "data": {
    // Response data
  },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-20T15:30:00Z"
  }
}
```

### Error Response

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The appointment slot is no longer available",
    "details": {
      "field": "appointment_slot",
      "reason": "slot_taken"
    }
  },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2024-01-20T15:30:00Z"
  }
}
```

## Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `AUTHENTICATION_FAILED` | 401 | Invalid or missing API key |
| `PERMISSION_DENIED` | 403 | Insufficient permissions for resource |
| `NOT_FOUND` | 404 | Resource not found |
| `INVALID_REQUEST` | 400 | Invalid request parameters |
| `RATE_LIMIT_EXCEEDED` | 429 | Too many requests |
| `INTERNAL_ERROR` | 500 | Server error |

## Quick Start Example

Here's a simple example to check appointment availability:

<CodeGroup>

```javascript Node.js
import { AvocaClient } from '@avoca/node-sdk';

const client = new AvocaClient({
  apiKey: process.env.AVOCA_API_KEY,
  workspaceId: process.env.WORKSPACE_ID
});

const availability = await client.scheduling.checkAvailability({
  serviceType: 'drain_cleaning',
  zipCode: '90210',
  dateRange: {
    start: '2024-01-20',
    end: '2024-01-25'
  }
});

console.log(availability.availableSlots);
```

```python Python
from avoca import AvocaClient
import os

client = AvocaClient(
    api_key=os.environ["AVOCA_API_KEY"],
    workspace_id=os.environ["WORKSPACE_ID"]
)

availability = client.scheduling.check_availability(
    service_type="drain_cleaning",
    zip_code="90210",
    date_range={
        "start": "2024-01-20",
        "end": "2024-01-25"
    }
)

print(availability["available_slots"])
```

```bash cURL
curl -X POST https://api.avoca.ai/v1/scheduling/availability \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "X-Workspace-ID: YOUR_WORKSPACE_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "service_type": "drain_cleaning",
    "zip_code": "90210",
    "date_range": {
      "start": "2024-01-20",
      "end": "2024-01-25"
    }
  }'
```

</CodeGroup>
